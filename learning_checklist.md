# AI Engineering Learning Checklist

## Phase 1: Core App Building
- [ ] Build a Python CLI todo app with save/load
- [ ] Build a FastAPI notes app with a simple HTML frontend
- [ ] Add SQLite or Postgres persistence to the notes app
- [ ] Add logging, error handling, and basic tests

## Phase 2: LLM App Basics
- [X] Build a simple chat app with streaming responses
- [X] Add conversation history and token counting
- [ ] Build a document Q&A app with embeddings and retrieval
- [ ] Build a structured output extractor that returns valid JSON

## Phase 3: Agent Foundations
- [ ] Build a tool-using agent with calculator/search/doc tools
- [ ] Build a multi-step research agent with planning and summaries
- [ ] Build a browser automation agent with Playwright

## Phase 4: Memory and Reliability
- [ ] Build short-term and long-term memory storage
- [ ] Add memory retrieval rules and expiration
- [ ] Build an eval set for one app you made
- [ ] Track quality, latency, token cost, and failure cases

## Phase 5: Production Systems
- [ ] Build an async inference queue with retries and rate limits
- [ ] Build a model router for cheap, fast, and smart models
- [ ] Build an observability dashboard for prompts and tool calls
- [ ] Build a nightly eval pipeline that catches regressions

## Phase 6: Advanced Topics
- [ ] Build a context engineering playground
- [ ] Build a memory benchmark to test recall quality
- [ ] Build a synthetic data or fine-tuning pipeline
- [ ] Build a mini AI app platform with users, sessions, tools, and evals

## Weekly Habits
- [ ] Ship one small version every week
- [ ] Write down what broke and what you learned
- [ ] Add tests after the first working version
- [ ] Measure latency and cost for every LLM feature
- [ ] Review and improve based on eval results
